# Hero Section Fix — Root Cause + Enhanced Visuals

## Context

The previous implementation looks flat despite having the right conceptual approach. Three root-cause bugs make everything render wrong:

### Bug #1: CSS Animation Destroys Inline Transforms (THE KILLER)
`animate-float-thesis` sets `transform: translateY(-14px)` which **completely replaces** the inline `transform: rotate(-15deg)`. CSS animation `transform` overrides — it does NOT compose. So once the float animation starts, ALL page rotations are lost and pages become upright flat rectangles.

### Bug #2: Triple Transform Conflict
Each page has THREE things fighting over the `transform` property:
1. Inline `style={{ transform: "rotate(-15deg)" }}`
2. Framer Motion `initial/animate` which writes its own `transform` (y, scale, opacity)
3. CSS class `animate-float-thesis` which writes `transform: translateY(...)`

Only ONE can win. Framer Motion wins over inline styles. Then CSS animation wins over both.

### Bug #3: Sphere Gradients Are Too Simple
A single `radial-gradient` produces a flat disc, not a glass orb. Real 3D glass needs:
- Multiple overlapping radial gradients (specular + body + edge darkening + bottom reflection)
- A pseudo-element for the bright specular highlight oval
- Better shadow layers

## Implementation

### Step 1: Fix Transform Architecture in `hero-section.tsx`

**Solution: Use Framer Motion for ALL transforms — drop CSS animation entirely.**

Framer Motion properly composes transforms. Give it `rotate`, `y`, `scale`, `rotateX`, `rotateY` as separate animated properties. It handles the math internally.

For the float effect, use Framer Motion's repeating animation instead of CSS `@keyframes`:

```tsx
// Each page uses Framer Motion for both entrance AND float
<motion.div
  className="absolute paper-page"
  style={{
    left: "4%",
    top: "10%",
    width: 230,
    zIndex: 1,
  }}
  // Rotation set as a constant (never changes after entrance)
  initial={{ opacity: 0, y: 40, scale: 0.88, rotate: -15 }}
  animate={{ opacity: 1, y: 0, scale: 1, rotate: -15 }}
  transition={{ duration: dur, delay: 0.5, ease: zenEase }}
>
  {/* Inner wrapper handles the float — pure translateY, no transform conflict */}
  <motion.div
    animate={{ y: [0, -14, 0] }}
    transition={{
      y: { duration: 6, repeat: Infinity, ease: "easeInOut" },
    }}
  >
    <ThesisPage ... />
  </motion.div>
</motion.div>
```

**Why nested divs?** Framer Motion's `animate` for a repeating float (`y: [0, -14, 0]`) and a one-shot entrance (`opacity: 0→1, y: 40→0`) can't coexist on the same element — the `y` values would conflict. Outer div: entrance animation + static rotation. Inner div: perpetual float loop.

Remove `animate-float-thesis` class from all pages entirely. The CSS keyframe and tailwind animation entry become dead code — remove them.

### Step 3: Enhance `GlassSphere` — Multi-Layer 3D Glass

Replace single radial-gradient with layered approach:

```tsx
function GlassSphere({ size, baseColor, darkColor, top, left, delay }: Props) {
  return (
    <motion.div
      className="absolute rounded-full"
      animate={{ y: [0, -12, 0] }}
      transition={{ y: { duration: 8, repeat: Infinity, ease: "easeInOut" } }}
      style={{
        width: size,
        height: size,
        top,
        left,
        animationDelay: delay,
        background: `
          radial-gradient(circle at 30% 22%, rgba(255,255,255,0.85) 0%, rgba(255,255,255,0.0) 36%),
          radial-gradient(circle at 50% 80%, rgba(255,255,255,0.12) 0%, transparent 50%),
          radial-gradient(circle at 50% 50%, rgba(${baseColor},0.7) 0%, rgba(${baseColor},0.85) 55%, rgba(${darkColor},0.95) 100%)
        `,
        boxShadow: `
          inset 0 -6px 16px -2px rgba(0,0,0,0.12),
          inset 0 2px 6px rgba(255,255,255,0.15),
          0 6px 20px -4px rgba(0,0,0,0.18),
          0 2px 6px -2px rgba(0,0,0,0.08)
        `,
        border: "1px solid rgba(255,255,255,0.3)",
      }}
    />
  );
}
```

Three gradient layers:
1. **Specular highlight** — bright white spot at 30% 22%, fades to transparent by 36%
2. **Bottom reflection** — subtle white glow at bottom
3. **Body colour** — base color in center, darker at edges (Fresnel-like)

Plus enhanced box-shadow:
1. **Inner bottom shadow** — darkens the bottom edge (away from light)
2. **Inner top highlight** — subtle white glow where light enters
3. **Outer shadow** — drop shadow beneath sphere
4. **Outer contact shadow** — tighter shadow near sphere
